About

griddy is a low-level grid-based CSS framework, inspired by the awesome Tailwind CSS and originally built as a capstone project for The Odin Project's HTML and CSS course. It's written in Sass/SCSS.

Quick Start

  1. Copy griddy.css or griddy.min.css into your project directory.
  2. Link the CSS file in the head of your page. <link rel="stylesheet" href="dist/griddy.min.css">
  3. After you built your page, consider running the purge command to remove unused classes to reduce filesize greatly.

Containers

Containers in griddy are just div elements with a fixed size which depend on your breakpoints. The max-width of .container-sm is the same as the min-width set by the sm breakpoint, for example. By default, those are:

.container-sm 640px

.container-md 768px

.container-lg 1024px

.container-xl 1280px

.container always at 100%

Flexbox

With the .flex class, you can turn a container into a flexbox. You can then apply flex properties to it and its items.

You can control flex-wrap with the .wrap, .wrap-none or .wrap-reverse classes.

In addition to .flex and .flex-column, there are also the .flex-reverse and .flex-column-reverse classe, which flip the flex items around.

Colors

griddy supports all named CSS colors by default. You can apply those to the background, text or border of your elements with the .bg-{color}, .text-{color} and .border-{color} properties. In addition, you can add your own colors to the $custom-colors array in _user.config.scss. Simply supply a name as a key and then a valid CSS color value (hex, rgb(a), or hsl all work) and you can use them just like the built-ins.

By applying pseudo-selectors, you can change the colors on different events too, for example on hover: or active: (click).

.bg-red .bg-greenyellow .bg-mistyrose .text-blue .bg-darkgoldenrod .bg-darkcyan .bg-fuchsia .hover:bg-darkkhaki .active:bg-oldlace .bg-cadetblue .text-cornsilk .border .border-firebrick

Typography

There are several classes for typography built into griddy. It's far from feature complete at this stage but it does support the most common properties. Note that some of the classes (like the extended weight classes) only work if your fonts support it.

Like the color classes, all typography classes also support pseudo-selectors.

.weight-lighter .weight-light .weight-regular .weight-bold .weight-bolder
.decoration-none .decoration-strikethrough .decoration-underline
.style-normal .style-italic .style-oblique .variant-caps
.align-left / .align-start .align-right / .align-end .align-center .align-justify

Height & Width

Height and width of elements can be configured with the .h-{val} and .w-{val} classes. There are many options as values available.

  • .w-100, .w-75, .w-50, .w-25 and .w-10 correspond to percentage values of the parent container element.
  • .w-100v, .w-75v, .w-50v, .w-25v and .w-10v correspond to view-width values of the viewport.
  • .w-100p, .w-200p, .w-300p, .w-400p and .w-500p correspond to exact pixel values.
  • .w-xs, .w-sm, .w-md, .w-lg and .w-lg depend on the default spacer value set in the config. By default, xs corresponds to $spacer * 1, sm to $spacer * 2, etc.
  • .w-0 sets width to none.

Height and width also support pseudo-selectors.

Margin & Padding

Margin and padding of elements can be configured with the .m{direction}-{val} and .p{direction}-{val} classes.

Valid values are 0 through 5, where 0 means no margin/padding, and the others depend on the spacer value set in the config. Margin also supports the auto value (you can use .mx-auto to center an element).

.p-0

No padding.

.p-1

.25 times spacer value, by default .25rem.

.p-2

.5 times spacer value, by default .5rem.

.p-3

Equal to spacer value, by default 1rem.

.p-4

1.5 times spacer value, by default 1.5rem.

.p-5

3 times spacer value, by default 3rem.

For directions, you can either just use p and m for all four directions (equivalent to just applying a value to the padding and margin properties, px / py for horizontal and vertical only, or pt / pr / pb / pl for top, right, bottom, or left respectively.

Note that these classes can override each other in order of appliance.
